Preguntas

  1. ¿Qué es el DOM?

    2. El DOM (Document Object Model) es una representación estructurada y jerárquica de un documento HTML/XML. Permite acceder y manipular los elementos del documento mediante scripting, como JavaScript. Es una interfaz que proporciona una forma de interactuar con los elementos del documento, agregar o eliminar contenido, cambiar estilos y realizar otras modificaciones dinámicamente.

  2. ¿Qué es un nodo?

    3. Un nodo es un elemento individual en una estructura jerárquica, como el DOM. Puede representar diferentes partes de un documento, como elementos HTML, atributos, texto, comentarios, entre otros. Los nodos se conectan entre sí formando un árbol y pueden tener relaciones padre-hijo, hermano-hermano o ancestro-descendiente. Cada nodo puede tener propiedades y métodos asociados para realizar operaciones en él.

  3. ¿Cómo se accede a un elemento del DOM?

    4. Se puede acceder a un elemento del DOM utilizando métodos proporcionados por el lenguaje de scripting, como JavaScript. Algunos métodos comunes incluyen:

    • getElementById(id): Accede a un elemento por su identificador único.
    • getElementsByClassName(className): Accede a una colección de elementos por su nombre de clase.
    • getElementsByTagName(tagName): Accede a una colección de elementos por su nombre de etiqueta.
    • querySelector(selector): Accede al primer elemento que coincide con el selector CSS especificado.
  4. ¿Cómo se modifica un elemento del DOM?

    5. Para modificar un elemento del DOM, se puede acceder al elemento deseado y utilizar métodos para cambiar su contenido, atributos, estilos, entre otros. Algunos métodos comunes incluyen:

    • innerHTML: Modifica el contenido HTML interno de un elemento.
    • setAttribute(name, value): Establece el valor de un atributo específico.
    • style.property = value: Modifica el estilo de un elemento cambiando una propiedad CSS específica.
    • appendChild(node): Agrega un nodo como hijo al elemento.
  5. ¿Cuál es la diferencia entre getElementby* y querySelector*?

    6. La diferencia principal entre los métodos getElementby* y querySelector* está en la forma en que se especifica el selector. getElementby* utiliza métodos que se enfocan en elementos específicos, como Id, ClassName y TagName, mientras que querySelector* utiliza selectores CSS para seleccionar elementos.

  6. ¿Qué es un evento?

    7. Un evento es una acción que ocurre en el navegador, como hacer clic en un elemento, mover el mouse, cargar una página, etc. Los eventos pueden ser detectados y utilizados para activar funciones o ejecutar código específico en respuesta a esa acción.

  7. ¿Qué es un listener?

    8. Un listener, también conocido como "event listener" o "manejador de eventos", es una función que se asocia a un elemento y espera la ocurrencia de un evento específico. Cuando el evento se dispara, el listener se activa y se ejecuta el código asociado a él. El listener permite detectar y responder a eventos, como hacer clic en un elemento, mover el mouse, presionar una tecla, entre otros. Al asociar un listener a un elemento, se puede definir qué acción realizar cuando ocurre el evento correspondiente.

  8. ¿Qué es un callback?

    9. En el contexto de los listeners y los eventos, un callback es una función que se pasa como argumento a otro método o función. Este callback se invoca cuando se produce un evento específico. Los callbacks se utilizan para definir la lógica que se ejecutará en respuesta a un evento. Por ejemplo, al hacer clic en un botón, se puede definir un callback que se ejecute para procesar esa acción.

  9. ¿Qué es un objeto de evento?

    10. Un objeto de evento contiene información y propiedades relacionadas con un evento específico que ha ocurrido en el DOM. Cuando se dispara un evento, se crea un objeto de evento que se pasa como argumento al callback asociado. Este objeto proporciona detalles como el tipo de evento, el elemento objetivo, las coordenadas del mouse, entre otros. Los objetos de evento se utilizan para obtener información y tomar decisiones basadas en el evento que ha ocurrido.
